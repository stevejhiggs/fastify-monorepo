# Temporal API

A Fastify API that starts and queries Temporal workflows. This API is part of a Temporal app generated by `pnpm generate:temporal-app`.

## Getting started

1. Start a Temporal server: `temporal server start-dev`
2. Start the worker (from the sibling `worker/` directory): `pnpm dev`
3. Start the API: `pnpm dev` (from this directory)
4. Visit `http://localhost:3000/health` to verify, or `http://localhost:3000/documentation` for Swagger docs

## Structure

```
src/
  index.ts              # Entry point
  listener.ts           # Network listener setup
  server.ts             # Fastify instance factory
  temporal-client.ts    # Lazy-initialized Temporal client singleton
  telemetry.ts          # OpenTelemetry instrumentation
  routes/
    route-registry.ts   # Central route registration
    health/route.ts     # Health check endpoint
    example-workflow/route.ts  # Example route that executes a Temporal workflow
```

## Route reference

| Method & Path         | Purpose                                                  |
| --------------------- | -------------------------------------------------------- |
| `GET /health`         | Returns `OK <uuid>` to confirm the service is healthy    |
| `POST /example-workflow` | Executes an example workflow and returns the result   |

## Environment Variables

- `LOG_LEVEL` - Logger level (default: `info`)
- `PORT` - Server port (default: `3000`)
- `DISABLE_DOCS` - Disable `/documentation` endpoint
- `TEMPORAL_ADDRESS` - Temporal server address (default: `localhost:7233`)
